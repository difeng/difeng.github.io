单向链表的反转

单向链表的反转

今天偶尔看到一个关于单链表反转的问题，说是用递归实现，提起了我的兴趣，记得当初校招找工作的时候，笔试经常会考，不过当时都是用迭代来做的，没有想过用递归来实现。以下是我的两种实现。

上述递归实现的思路如下：
从当前节点依次递归反转下一个节点，
每次递归的出口条件是下个节点的next为null，
当前节点后面的所有节点都反转成功后，将此当前节点的next也设置为null，为上层递归做好出口。

java线程池详解

1. 为什么要引入线程池技术

在多线程环境下，线程的创建和销毁花费的时间和消耗的系统资源都是相当大的，如果有太多的线程创建，而不能对其加以控制，那么最终会导致系统资源不足，为了尽可能减少的创建和销毁线程的次数，充分利用已有资源来进行服务，因此引入了线程池。

2. 合理使用线程池的好处

（1）降低资源消耗。通过重复利用已创建的线程降低线程创建和销毁造成的消耗
（2）提高响应速度。当任务到达时，任务可以不需要等到线程创建就能立即执行。
（3）提高线程的可管理性。线程是稀缺资源，如果无限制的创建，不仅会消耗系统资源，还会降低系统的稳定性，使用线程池可以进行统一的分配，调优和监控。但是要做到合理的利用线程池，必须对其原理了如指掌。

3. 线程池工作原理

一般来说一个线程池包括如下几个组成部分：
（1）线程管理器（ThreadPool）
   用于创建并管理线程池
（2）工作线程（PoolWorker）
   线程池中实际执行任务的线程
（3）任务接口（Task）
   每个任务必须实现的接口，以供工作线程调度任务的执行，它主要规定了任务的入口，任务执行完后的收尾工作，任务的执行状态等；
（4）任务队列（taskqueue）
   用于存放没有处理的任务

4. java中线程池的使用

（1）线程池创建的几个方式
使用Executors提供的几个静态方法来创建
public static ExecutorService newSingleThreadExecutor()
public static ExecutorService newFixedThreadPool(int nThreads)
public static ExecutorService newCachedThreadPool()
public static ScheduledExecutorService newScheduledThreadPool(int corePoolSize)
newSingleThreadExcutor():
返回一个包含单线程的线程池，该线程池只能含有一个线程，当该线程执行过程有异常或故障，将会启动一个新的线程继续执行该线程未完成的任务。
ExecutorService newFixedThreadPool(int nThreads)：
返回一个包含指定数目线程的线程池。
public static ExecutorService newCachedThreadPool()
创建一个可重用其中线程的线程池，即根据需要创建线程，若线程池中有可重用的线程将调用之前构造的线程，若没有则新建一个线程并将其添加到线程池u中。尚未使用超过60秒的线程将会从线程池中移除。
newScheduledThreadPool(int corePoolSize)：
创建一个可以进行定时或周期执行任务的线程池。
（2）使用线程池和不使用线程池的对比

执行结果：
没有使用线程池前:
1000 个线程没有使用线程池花费 103 毫秒
使用线程池后:
1000 个线程使用线程池花费 18 毫秒

明显可以看出使用线程池后花费的时间更少
